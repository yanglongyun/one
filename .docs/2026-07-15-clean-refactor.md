# 0.4.0 干净重构记录

日期：2026-07-15

## 本次原则

- 只保留一个现行协议、一个数据库结构和一套业务实现，不提供旧字段、旧工具名、旧客户端或旧表结构的兼容路径。
- 视觉模型、截图能力及其前端、服务端、桌面端、安卓端配置已经删除，不以隐藏开关或空实现保留。
- `worker/schema.sql` 是数据库唯一真相；代码运行时不创建或修补平台系统表。
- 重构先在个人版完成，基础检查通过后同步到开源版；没有部署，也没有发布安装包。

## 已完成的结构性修改

### Agent 与工具

- 浏览器工具统一为 `chrome_debugger`，明确通过 Chrome 扩展的 `chrome.debugger.sendCommand` 操作当前活动标签页。
- 删除 `browser_cdp`、视觉和截图工具；提示词、执行分发和各客户端能力声明同步删除。
- 新增单一管理工具 `one_manage`，记忆、目标、日程、任务和小应用都通过正式业务仓储修改。
- `sql` 只允许读取平台数据或写入 `app_*` 小应用数据表，禁止直接修改平台系统表。
- 主对话、压缩、小应用桥和模型连接检查共用同一模型请求构造逻辑。

### 任务、目标与日程

- 日程到期、目标推进、目标验收的任务创建和状态推进改为 D1 原子批次。
- 目标推进完成后创建独立验收任务；未通过时自动安排下一轮，完成或需要用户时才停止。
- 任务增加租约、恢复和最多三次执行尝试；DO 只维护一个最近到期 alarm，不再使用队列、Cron 或全量每分钟唤醒。
- 删除会话或取消任务时先终止并等待对应 Agent，未停止时返回冲突，不提前删除数据。

### 实时连接

- 执行端协议固定为版本 1，桌面、安卓和浏览器客户端版本统一为 0.4.0。
- 不接受旧协议；旧客户端直接收到升级错误并停止重连。
- 工具结果以 `threadId + callId` 关联，避免并发会话中相同调用编号互相覆盖。
- 多标签页只接收自己发起的流，其他标签页完成后刷新历史，不再串流。
- 浏览器手动断开后不会被保活 alarm 自动重新连接；配置变更会按明确连接状态重连。

### 数据和界面

- 会话、任务、目标、日程、记忆都改为服务端游标分页，不再存在固定数量截断。
- 设置页使用顶部导航，删除“云端服务正常”状态块，并与其他业务页面共用 `page-inner` 宽度。
- 对话开始摘要阈值统一为 64000 tokens。
- 小应用元信息和代码文件统一经过仓储校验；只允许 `index.html`、`index.js`、`index.css`、`index.sql`。

### 下载发布

- 下载入口改为 R2 发布清单，当前版本路径固定为 `releases/0.4.0/`。
- macOS、Windows、Android、浏览器四个平台只有在对应对象真实存在时才显示下载按钮。
- 本次没有把旧包伪装成 0.4.0，也没有上传调试包。

## 上线前必须执行

1. 备份需要保留的数据。
2. 使用当前 `worker/schema.sql` 创建全新的 `one` D1 结构；不要继续使用缺少任务租约、消息客户端编号等字段的旧库。
3. 生成并签名 0.4.0 正式安装包，按下载清单中的精确文件名上传到 R2 桶 `one`。
4. 部署 Worker 后由产品负责人自行完成真实业务测试。
